What is VirtualStage

VirtualStage is a 3D visualization tool for designing LED screen and projection surface layouts. It lets you create, position, rotate and scale multiple screens in a 3D space, and receive real-time video from applications like Resolume Arena via Syphon (macOS) or Spout (Windows).

Use Cases

  • Previewing stage setups with LED screens before the actual build
  • Designing multi-screen layouts for live shows
  • Testing video content across different screen configurations
  • Directly importing Resolume Advanced Output presets

Installation

macOS

  1. Download VirtualStage-macOS.zip from the releases page
  2. Unzip the file
  3. Right-click the app and select Open (first time only)
  4. If macOS says the app is damaged, open Terminal and run: 
    xattr -cr /path/to/VirtualStage.app
  5. Open the app normally

Windows

  1. Download VirtualStage-Windows.zip from the releases page
  2. Unzip to your desired folder
  3. Run VirtualStage.exe

Getting Started

On first launch, an authentication screen appears where you can:

  • Sign in with an existing email and password
  • Register a new account (email confirmation required)

Your account enables cloud project saving and preference syncing across devices.

Once authenticated, the editor opens with a default virtual screen in the center of the scene.

Interface

Menu Bar

  • File: New, Open, Save, Save As, Cloud Save/Load, Autosave, Preferences, Log Out, Quit
  • View: Show/hide properties panel sections (Ambient Light, Position, Rotation, Size, Input Mapping)
  • Link: Import layout from Resolume (Input or Output)
  • Help: Manual, Check for Updates, About

Sidebar (Left Panel)

  • SCREENS: All project screens. Click to select, [X] to delete
  • SERVERS: Available video sources (Syphon/Spout). Click a server to assign it to the selected screen

The sidebar scrolls if the content exceeds the window height.

3D Viewport

The central area where screens are displayed in 3D. Here you can:

  • Orbit the camera (left-click + drag on empty area)
  • Zoom (mouse scroll)
  • Pan (middle-click + drag)
  • Select screens (left-click on a screen)
  • Manipulate with the gizmo (drag gizmo axes)

Properties Panel (Right)

Appears when a screen is selected. Shows and allows editing:

  • Position (X, Y, Z)
  • Rotation (X, Y, Z in degrees)
  • Size (Width, Height)
  • Curvature (arc degrees, -180 to 180)
  • Input Mapping (Crop X, Y, W, H)
  • Ambient Light (scene ambient lighting)
Right-click on any parameter slider to type an exact value.

Toolbar

Centered at the bottom, shows the three gizmo modes. The active mode is highlighted in blue.

Status Bar

Shows current mode (DESIGNER/VIEW), project name, FPS, server count, and available shortcuts.

Work Modes

Toggle between modes with Tab.

Designer Mode

The main editing mode. Includes the 3D reference grid, transformation gizmo, sidebar, properties panel, toolbar, and full menu bar.

View Mode

Clean preview mode. Hides all UI elements except a minimal status bar. Ideal for seeing the final design result.

  • The window automatically goes always-on-top
  • The menu bar is hidden
  • Lock the camera with the padlock button (top-right corner)
  • Camera presets available with keys 1, 2, 3, 0

Virtual Screens

Add a Screen

Press A or use the [A]dd button in the SCREENS header. A new screen (320x180) is created at the center.

Select Screens

  • Left-click on a screen in the viewport or sidebar
  • Cmd+Click (macOS) or Ctrl+Click (Windows) to add/remove from selection
  • Shift+Click in the sidebar for range selection
  • Press S to enable area selection mode, then drag a rectangle

Delete Screens

Select one or more screens and press Delete or Backspace. You can also use the [X] button in the sidebar.

Duplicate a Screen

Right-click on a screen and select Duplicate. The copy is created with a position offset and keeps the assigned video source.

Screen Types

  • Flat: Standard rectangular screen (default)
  • Curved: Cylindrical arc screen. Activated by changing the Curvature parameter (-180 to +180 degrees)
  • Polygon: Non-rectangular masked screen. Created when importing polygons from Resolume

Transforming Screens

Screens are manipulated with the Gizmo, a 3D widget that appears on the selected screen.

KeyModeDescription
WTranslateDrag red (X), green (Y), or blue (Z) axes to move
ERotateDrag rings to rotate around each axis
RScaleDrag to resize the screen

When multiple screens are selected, the gizmo appears on the primary screen and the transformation applies to all selected screens.

Right-click on any slider in the properties panel to enter an exact numeric value.

Video Sources

VirtualStage receives real-time video textures from other applications:

  • macOS: Via Syphon protocol (Resolume, MadMapper, VDMX, etc.)
  • Windows: Via Spout protocol (Resolume, TouchDesigner, OBS, etc.)

Assign a Source

  1. Sidebar: Select the screen, then click a server in the SERVERS list
  2. Number keys: Select the screen and press 1-9
  3. Context menu: Right-click on the screen and pick a server

Disconnect a Source

Select the screen and press D, or right-click > Disconnect.

Input Mapping Editor

The Input Mapping editor lets you crop which portion of the video texture is displayed on each screen.

Open the Editor

Select a screen and press M, or right-click > Map.

Editor Interface

Shows the video texture fullscreen (or a checkerboard pattern if no texture). A yellow rectangle represents the crop area.

  • Faint grid lines at 5% intervals; stronger lines at 25%, 50%, 75%
  • Crop values (X, Y, W, H) shown below, normalized 0-1

Editor Controls

Import from Resolume

VirtualStage can import layouts directly from Resolume Arena Advanced Output XML files.

How to Import

  1. Press L (or use the Link menu)
  2. If screens already exist, confirm by pressing L again
  3. Choose rectangle type:
    • I — Input coordinates (content)
    • O — Output coordinates (projection)
  4. VirtualStage automatically finds the most recent XML in the Resolume presets folder
  5. If not found, a file dialog opens for manual selection

What Gets Imported

  • Each Slice becomes a flat screen with proportional size and position
  • Each Polygon becomes a polygon-masked screen
  • Crop regions (Input Mapping) are calculated automatically for each screen
  • Screens are scaled and distributed in 3D space maintaining the original layout proportions

Que es VirtualStage

VirtualStage es una herramienta de visualizacion 3D para disenar layouts de pantallas LED y superficies de proyeccion virtuales. Permite crear, posicionar, rotar y escalar multiples pantallas en un espacio 3D, y recibir video en tiempo real desde aplicaciones como Resolume Arena mediante Syphon (macOS) o Spout (Windows).

Casos de uso

  • Previsualizacion de escenarios con pantallas LED antes del montaje real
  • Diseno de layouts multi-pantalla para shows en vivo
  • Prueba de contenido de video en diferentes configuraciones de pantallas
  • Importacion directa de presets de Resolume Advanced Output

Instalacion

macOS

  1. Descarga el archivo VirtualStage-macOS.zip desde la pagina de releases
  2. Descomprime el archivo
  3. La primera vez, haz clic derecho sobre la app y selecciona Abrir
  4. Si macOS dice que la app esta danada, abre Terminal y ejecuta: 
    xattr -cr /ruta/a/VirtualStage.app
  5. Abre la app normalmente

Windows

  1. Descarga el archivo VirtualStage-Windows.zip desde la pagina de releases
  2. Descomprime en la carpeta deseada
  3. Ejecuta VirtualStage.exe

Primer inicio

Al abrir VirtualStage por primera vez, aparece una pantalla de autenticacion donde puedes:

  • Iniciar sesion con un email y contrasena existentes
  • Registrar una cuenta nueva (se requiere confirmacion por email)

La cuenta permite guardar y cargar proyectos en la nube y sincronizar preferencias entre dispositivos.

Una vez autenticado, se abre el editor con una pantalla virtual por defecto en el centro de la escena.

Interfaz

+-------------------------------------------------------+ | File View Link Help [Autosave ON] | Barra de menu +----------+--------------------------------------------+ | | | | SCREENS | | | Screen1 | Vista 3D | | Screen2 | (Viewport) | | | | |----------| | | SERVERS | | | Source1 | | | Source2 | [W] Move [E] Rotate [R] Scale | Toolbar +----------+--------------------------------------------+ | DESIGNER | project.json | FPS: 60 | Servers: 2 | ... | Barra de estado +-------------------------------------------------------+

Barra de menu

  • File: Nuevo, Abrir, Guardar, Guardar Como, Guardar/Cargar en la Nube, Autosave, Preferencias, Cerrar sesion, Salir
  • View: Muestra u oculta secciones del panel de propiedades (Ambient Light, Position, Rotation, Size, Input Mapping)
  • Link: Importar layout desde Resolume (Input u Output)
  • Help: Manual, Buscar actualizaciones, Acerca de

Panel lateral (Sidebar)

  • SCREENS: Todas las pantallas del proyecto. Click para seleccionar, [X] para eliminar
  • SERVERS: Fuentes de video disponibles (Syphon/Spout). Click en un servidor para asignarlo a la pantalla seleccionada

El panel lateral tiene scroll si el contenido excede el alto de la ventana.

Vista 3D (Viewport)

El area central donde se visualizan las pantallas en 3D. Aqui se puede:

  • Orbitar la camara (click izquierdo + arrastrar en area vacia)
  • Hacer zoom (scroll del mouse)
  • Pan (click medio + arrastrar)
  • Seleccionar pantallas (click izquierdo sobre una pantalla)
  • Manipular con el gizmo (arrastrar los ejes del gizmo)

Panel de propiedades (Derecha)

Aparece cuando hay una pantalla seleccionada. Muestra y permite editar:

  • Position (X, Y, Z)
  • Rotation (X, Y, Z en grados)
  • Size (Ancho, Alto)
  • Curvature (grados de arco, -180 a 180)
  • Input Mapping (Crop X, Y, W, H)
  • Ambient Light (iluminacion ambiental de la escena)
Haz clic derecho sobre cualquier slider para escribir un valor exacto.

Toolbar

Centrada en la parte inferior, muestra los tres modos del gizmo. El modo activo se resalta en azul.

Barra de estado

Muestra el modo actual (DESIGNER/VIEW), nombre del proyecto, FPS, cantidad de servidores y atajos disponibles.

Modos de trabajo

Alterna entre modos con Tab.

Modo Designer

El modo de edicion principal. Incluye la grilla 3D de referencia, gizmo de transformacion, panel lateral, panel de propiedades, toolbar y barra de menu completa.

Modo View

Modo de previsualizacion limpia. Oculta todos los elementos de UI excepto la barra de estado minima. Ideal para ver el resultado final del diseno.

  • La ventana se coloca automaticamente "siempre al frente"
  • La barra de menu se oculta
  • Se puede bloquear la camara con el boton de candado (esquina superior derecha)
  • Presets de camara disponibles con teclas 1, 2, 3, 0

Pantallas virtuales

Agregar una pantalla

Presiona A o usa el boton [A]dd en el encabezado SCREENS. Se crea una nueva pantalla (320x180) en el centro.

Seleccionar pantallas

  • Click izquierdo sobre una pantalla en el viewport o en el sidebar
  • Cmd+Click (macOS) o Ctrl+Click (Windows) para agregar/quitar de la seleccion
  • Shift+Click en el sidebar para seleccion por rango
  • Presiona S para activar modo de seleccion por area, luego arrastra un rectangulo

Eliminar pantallas

Selecciona una o mas pantallas y presiona Delete o Backspace. Tambien puedes usar el boton [X] en el sidebar.

Duplicar una pantalla

Haz clic derecho sobre una pantalla y selecciona Duplicate. La copia se crea con un offset de posicion y mantiene la fuente de video asignada.

Tipos de pantalla

  • Flat: Pantalla plana rectangular (por defecto)
  • Curved: Pantalla curva cilindrica. Se activa al modificar el parametro Curvature (-180 a +180 grados)
  • Polygon: Pantalla con mascara poligonal no rectangular. Se crea al importar poligonos desde Resolume

Transformar pantallas

Las pantallas se manipulan con el Gizmo, un widget 3D que aparece sobre la pantalla seleccionada.

TeclaModoDescripcion
WTranslate (Mover)Arrastra los ejes rojo (X), verde (Y) o azul (Z)
ERotate (Rotar)Arrastra los anillos para rotar alrededor de cada eje
RScale (Escalar)Arrastra para cambiar el tamano

Cuando hay multiples pantallas seleccionadas, el gizmo se muestra en la pantalla principal y la transformacion se aplica a todas.

Haz clic derecho sobre cualquier slider del panel de propiedades para ingresar un valor exacto.

Fuentes de video

VirtualStage recibe texturas de video en tiempo real desde otras aplicaciones:

  • macOS: Via protocolo Syphon (Resolume, MadMapper, VDMX, etc.)
  • Windows: Via protocolo Spout (Resolume, TouchDesigner, OBS, etc.)

Asignar una fuente

  1. Sidebar: Selecciona la pantalla, luego haz click en un servidor de la lista SERVERS
  2. Teclas numericas: Selecciona la pantalla y presiona 1-9
  3. Menu contextual: Clic derecho sobre la pantalla y selecciona un servidor

Desconectar una fuente

Selecciona la pantalla y presiona D, o clic derecho > Disconnect.

Editor de Input Mapping

El editor permite recortar que porcion de la textura de video se muestra en cada pantalla.

Abrir el editor

Selecciona una pantalla y presiona M, o clic derecho > Map.

Interfaz del editor

Muestra la textura de video a pantalla completa (o un patron cuadriculado si no hay textura). Un rectangulo amarillo representa el area de recorte (crop).

  • Lineas tenues al 5% y lineas mas visibles al 25%, 50% y 75%
  • Valores del crop (X, Y, W, H) debajo, normalizados 0-1

Controles del editor

Redimensionar por esquinaArrastrar una de las 4 esquinas
Redimensionar por bordeArrastrar uno de los 4 bordes centrales

El snap alinea los bordes a incrementos del 5%, facilitando la alineacion entre multiples pantallas.

Importar desde Resolume

VirtualStage puede importar layouts directamente desde archivos XML de Resolume Arena Advanced Output.

Como importar

  1. Presiona L (o usa el menu Link)
  2. Si ya hay pantallas, confirma presionando L nuevamente
  3. Elige el tipo de rectangulo:
    • I — Coordenadas de Input (contenido)
    • O — Coordenadas de Output (proyeccion)
  4. VirtualStage busca automaticamente el XML mas reciente en la carpeta de presets de Resolume
  5. Si no lo encuentra, se abre un dialogo para seleccionar el archivo manualmente

Que se importa

  • Cada Slice se convierte en una pantalla plana con tamano y posicion proporcional
  • Cada Polygon se convierte en una pantalla con mascara poligonal
  • Se calculan automaticamente las regiones de crop para cada pantalla
  • Las pantallas se escalan y distribuyen en el espacio 3D manteniendo las proporciones del layout original

Proyectos

Nuevo proyecto

File > New Project limpia todas las pantallas y crea una por defecto.

Guardar

  • Cmd+S / Ctrl+S: Guarda en la ubicacion actual (abre dialogo si es proyecto nuevo)
  • Cmd+Shift+S / Ctrl+Shift+S: Guardar como... (nueva ubicacion)

Los proyectos se guardan como archivos .json con toda la informacion de pantallas, transformaciones, fuentes, crop y estado de camara.

Abrir

Cmd+O / Ctrl+O. Restaura todas las pantallas, la posicion de camara, e intenta reconectar las fuentes de video. El autosave se activa automaticamente.

Autosave

  • Activa/desactiva desde File > Autosave (15s)
  • Guarda automaticamente cada 15 segundos (local o nube)
  • Se activa automaticamente al abrir un proyecto existente

Deshacer / Rehacer

  • Cmd+Z / Ctrl+Z: Deshacer
  • Cmd+Shift+Z / Ctrl+Shift+Z: Rehacer

Almacenamiento en la nube

Guardar en la nube

File > Save to Cloud. Si es un proyecto nuevo, se pide un nombre. La subida ocurre en segundo plano.

Cargar desde la nube

File > Load from Cloud. Muestra una lista de proyectos guardados con fechas. Click para cargar, Esc para cerrar.
